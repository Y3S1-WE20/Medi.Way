```java
public interface ITestListener {
    void onStart(ITestContext context);           // Suite starts
```

## 📋 pom.xml Configuration for Reporting

### Maven Surefire Plugin

**Location:** `pom.xml` lines 122-146

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.2.5</version>
    <configuration>
        <suiteXmlFiles>
            <suiteXmlFile>testng.xml</suiteXmlFile>
        </suiteXmlFiles>
        
        <!-- Enable default TestNG listeners -->
        <properties>
            <property>
                <name>usedefaultlisteners</name>
                <value>true</value>  <!-- Enables HTML/XML report generation -->
            </property>
        </properties>
        
        <!-- Custom output directory -->
        <reportsDirectory>${project.build.directory}/test-reports</reportsDirectory>
    </configuration>
    
    <dependencies>
        <dependency>
            <groupId>org.apache.maven.surefire</groupId>
            <artifactId>surefire-testng</artifactId>
            <version>3.2.5</version>
        </dependency>
    </dependencies>
</plugin>
```

**Configuration Breakdown:**

1. **`<usedefaultlisteners>true</usedefaultlisteners>`**
   - Enables TestNG's built-in HTML/XML report generation
   - Without this: Only console output, no HTML reports
   - With this: `index.html`, `emailable-report.html`, `testng-results.xml` generated

2. **`<reportsDirectory>target/test-reports</reportsDirectory>`**
   - Custom output folder (default is `target/surefire-reports`)
   - Easier to find reports in Jenkins/CI
   - Consistent path across different CI systems

3. **`<surefire-testng>` dependency**
   - Forces Maven to recognize TestNG tests
   - Without this: Surefire might pick JUnit provider → 0 tests run

## 🔌 Listener Registration in testng.xml

**Location:** `testng.xml` lines 5-7

```xml
<suite name="MEDI.WAY Test Suite">
    <listeners>
        <listener class-name="backend.listeners.MediWayTestListener"/>
    </listeners>
    <!-- ... tests ... -->
</suite>
```

**How It Works:**

1. Maven runs `mvn test`
2. Surefire plugin reads `testng.xml`
3. TestNG sees `<listener>` tag
4. Instantiates `MediWayTestListener` class
5. Hooks listener methods into test lifecycle
6. Listener prints custom output during test execution

**Alternative Registration (not used):**

```java
// In test class (less flexible)
@Listeners(MediWayTestListener.class)
public class AssertionsDemoTest { }
```

**Why testng.xml registration is better:**
- ✅ Centralized: One place for all listeners
- ✅ Flexible: Easy to enable/disable
- ✅ Reusable: Same listener for all test classes
- ✅ No code changes: Just edit XML

#### 5️⃣ **onTestFailure() - Detailed Failure Reporting**

**Location:** Lines 127-160

```java
@Override
public void onTestFailure(ITestResult result) {
    failedTests++;
    long duration = result.getEndMillis() - result.getStartMillis();
    
    String testName = result.getMethod().getMethodName();
    Throwable error = result.getThrowable();
    
    failedList.add(new TestResultInfo(className, testName, duration, error));
    
    log(String.format("  ❌ FAILED: %s (%dms)", testName, duration));
    
    // Print detailed error information
    if (error != null) {
        log("  └─ Error: " + error.getClass().getSimpleName());
        log("  └─ Message: " + error.getMessage());
        
        // Print stack trace (truncated)
        StackTraceElement[] stackTrace = error.getStackTrace();
        if (stackTrace.length > 0) {
            log("  └─ Location: " + stackTrace[0]);
        }
    }
}
```

**Explain to Audience:**
> "When a test fails, `onTestFailure()` provides rich debugging information:
> - Red X icon ❌ for visual identification
> - Test duration (was it a timeout?)
> - Error type and message
> - Stack trace location
> 
> This saves developers time - they immediately see WHERE and WHY the test failed without digging through logs."

**Business Value:**
- **Faster debugging:** Error details right in console
- **Root cause analysis:** Stack traces pinpoint exact failure location
- **Time savings:** Developers don't need to search through thousands of log lines

#### 6️⃣ **onFinish() - Comprehensive Summary Report**

**Location:** Lines 177-260

```java
@Override
public void onFinish(ITestContext context) {
    long totalTime = System.currentTimeMillis() - startTime;
    int totalTests = passedTests + failedTests + skippedTests;
    double passRate = totalTests > 0 ? (passedTests * 100.0 / totalTests) : 0;
    
    StringBuilder summary = new StringBuilder();
    
    // Summary Header
    summary.append("╔══════════════════════════════════════════════════════════╗\n");
    summary.append("║             TEST EXECUTION SUMMARY                       ║\n");
    summary.append("╠══════════════════════════════════════════════════════════╣\n");
    summary.append(String.format("║  📊 STATISTICS                                           ║\n"));
    summary.append(String.format("║  ├─ Total Tests:     %-40s  ║\n", totalTests));
    summary.append(String.format("║  ├─ ✅ Passed:       %-40s  ║\n", passedTests));
    summary.append(String.format("║  ├─ ❌ Failed:       %-40s  ║\n", failedTests));
    summary.append(String.format("║  ├─ ⏭️  Skipped:      %-40s  ║\n", skippedTests));
    summary.append(String.format("║  ├─ Pass Rate:       %-40s  ║\n", String.format("%.2f%%", passRate)));
    summary.append(String.format("║  └─ Duration:        %-40s  ║\n", formatDuration(totalTime)));
    
    // Progress Bar (visual representation)
    int barWidth = 40;
    int passedWidth = (int) ((passedTests * barWidth) / (double) totalTests);
    int failedWidth = (int) ((failedTests * barWidth) / (double) totalTests);
    
    String progressBar = "█".repeat(passedWidth) + 
                         "▓".repeat(failedWidth) +
                         "░".repeat(barWidth - passedWidth - failedWidth);
    
    summary.append("║  Progress: [" + progressBar + "]                         ║\n");
    
    log(summary.toString());
    
    // Generate HTML Report
    generateHtmlReportSnippet(totalTests, passedTests, failedTests, skippedTests, totalTime);
}
```

**Explain to Audience:**
> "After all tests complete, `onFinish()` generates a beautiful summary box with:
> - **Total test count and breakdown** (passed/failed/skipped)
> - **Pass rate percentage** (key metric for quality gates)
> - **Total execution time** (suite performance)
> - **Visual progress bar** using Unicode box characters
> 
> This summary is perfect for:
> - CI/CD pipeline reports
> - Stakeholder presentations
> - Quality metrics dashboards"

**Business Value:**

```java
private void generateHtmlReportSnippet(int total, int passed, int failed, int skipped, long duration) {
    try {
        java.nio.file.Path outputDir = java.nio.file.Paths.get("target/test-reports");
        java.nio.file.Files.createDirectories(outputDir);
        
        PrintWriter htmlWriter = new PrintWriter(new FileWriter("target/test-reports/mediway-summary.html"));
        
        htmlWriter.println("<!DOCTYPE html>");
        htmlWriter.println("<html><head><title>MEDI.WAY Test Summary</title>");
        htmlWriter.println("<style>");
        htmlWriter.println("body { font-family: Arial; background: #f5f5f5; }");
        htmlWriter.println(".stat { text-align: center; padding: 15px; border-radius: 8px; }");
        htmlWriter.println(".stat.passed { background: #d4edda; color: #155724; }");
        htmlWriter.println("</style></head><body>");
        
        // Statistics boxes
        htmlWriter.printf("<div class='stat passed'><div class='number'>%d</div>Passed</div>", passed);
        
        // Progress bar
        double passedPct = total > 0 ? (passed * 100.0 / total) : 0;
        htmlWriter.printf("<div class='progress-bar' style='width: %.1f%%'></div>", passedPct);
        
        htmlWriter.println("</body></html>");
        htmlWriter.close();
        
        System.out.println("\n📄 Custom HTML report generated: target/test-reports/mediway-summary.html\n");
    } catch (IOException e) {
        System.err.println("Warning: Could not create HTML summary");
    }
}
```

**Explain to Audience:**
> "Beyond console output, we generate a custom HTML summary page with:
> - Color-coded statistics boxes (green for passed, red for failed)
> - Visual progress bar
> - Professional styling for stakeholder viewing
> 
> This HTML file can be:
> - Attached to emails
> - Embedded in CI/CD dashboards
> - Archived for compliance audits"
