# Docker Setup for Medi.Way

This guide explains how to build and run the Medi.Way backend (Spring Boot) and frontend (React) using Docker and Docker Compose.

## Components

- **backend**: Spring Boot application (port 8080)
- **frontend**: React static build served by nginx (port 3000 mapped to internal 80)
- **db**: MySQL 8 database (port 3306) — optional; for local development you can keep using the H2 test profile if preferred.

## Prerequisites

- Docker Engine + Docker Compose Plugin (Docker Desktop on macOS covers both)
- Ports 3000, 8080, and 3306 available locally

## Build & Run All Services

```bash
docker compose up --build
```

Then access:

- Frontend: http://localhost:3000
- Backend API: http://localhost:8080
- MySQL: localhost:3306 (use credentials in `docker-compose.yml`)

Stop services:
```bash
docker compose down
```

Remove volumes (clear database data):
```bash
docker compose down -v
```

## Environment Configuration

The backend reads standard Spring Boot datasource env vars defined in `docker-compose.yml`:

- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`
- `SPRING_PROFILES_ACTIVE`

For development without MySQL, you can switch to the H2 test profile:

```yaml
environment:
  SPRING_PROFILES_ACTIVE: test
```

Frontend build accepts an API base URL via build arg:

```yaml
build:
  args:
    REACT_APP_API_BASE_URL: http://localhost:8080
```

If you need a different backend endpoint (e.g., behind a reverse proxy), change that arg and rebuild.

## Rebuilding After Code Changes

Because of multi-stage builds, rebuild only the changed service:

```bash
docker compose build backend
docker compose build frontend
docker compose up -d
```

## Running Backend Alone

```bash
cd backend
docker build -t mediway-backend .
docker run --rm -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=test mediway-backend
```

## Running Frontend Alone (Against Local Backend)

Ensure backend is running on `localhost:8080`, then:

```bash
cd frontend
docker build -t mediway-frontend .
docker run --rm -p 3000:80 mediway-frontend
```

## Common Adjustments

- **JVM memory**: Tweak `JAVA_OPTS` in compose for production sizing.
- **Database persistence**: Volume `db_data` keeps data between restarts.
- **HTTPS**: Add a reverse proxy (Caddy/Traefik/Nginx) or terminate TLS in a load balancer.
- **Front-end caching**: Nginx config (`frontend/nginx.conf`) sets 30d cache for `/static/` assets.

## Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| Backend fails to start | DB not ready | Increase `depends_on`, add healthcheck, or use H2 profile |
| Frontend 404 on refresh | SPA route not served | Nginx `try_files` already configured; ensure `nginx.conf` copied |
| DB auth errors | Wrong credentials | Match compose env vars with Spring datasource settings |
| CORS issues | Missing CORS config | Add CORS mappings to Spring configuration or proxy requests |
